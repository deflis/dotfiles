# コミットメッセージの書き方

- 変更が1つの場合は簡潔な1行にまとめること。複数行にはしない。
- 日本語の体言止めで書くこと。
- 変更内容がわかるように、可能な限りコンポーネント名やファイル名、関数名などを含めること。
- 変更が複数ある場合は、最初の行に概要を1行で書き、その後に箇条書きで詳細を書くこと。
- Commit type などの Prefix は絶対に付けないこと。今回の修正の変更内容を簡潔にまとめる。
  - モノレポでも Prefix は不要で、例にあるような簡潔なものを求めます。

1行のコミットメッセージの例:
`Buttonコンポーネントにdisabledプロパティを追加`

複数行コミットメッセージの例:

```
Headerコンポーネントのナビゲーションを修正

- リンク先に`/link`を追加
- リンク先に追加に合わせてスタイルを更新
```
